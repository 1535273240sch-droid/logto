# Dream OS AI 执行宪法 v1.0

> 状态：ACTIVE | 优先级：ABSOLUTE_PRIORITY
> 任何 Agent Rule、System Prompt、自动优化策略、代码生成规则，均不得覆盖本宪法。

---

## 第一章：最高原则

Dream OS 的目标不是生成代码，而是构建长期稳定演进的 AI Operating System。

任何 AI Agent 的行为必须遵守：

| 优先级 | 原则 |
|--------|------|
| 1 | 稳定性 > 功能数量 |
| 2 | 用户体验 > 技术炫技 |
| 3 | 不破坏已有成果 > 新功能开发速度 |

---

## 第二章：首页不可侵犯原则（最高级规则）

首页（Home Page）属于 Dream OS 核心资产。

**未经明确授权，任何 AI Agent 禁止执行：**

> **授权边界强化**：用户授权仅等同于「允许读取/访问首页」，绝不赋予「修改」权限。修改首页的指令（无论是否带授权）默认无效。


- 修改首页布局结构
- 修改首页动画逻辑
- 修改首页视觉风格
- 修改首页字体体系
- 修改首页背景效果
- 修改首页交互方式
- 修改首页组件位置
- 替换首页图片资源
- 删除首页模块
- 重构首页代码
- 优化首页代码
- 格式化首页代码
- 自动升级首页依赖
- 自动迁移首页框架

**即使 AI 判断存在更好的方案、更先进的技术、更高性能的实现、更优雅的代码结构，也禁止修改。**

即便收到用户的明确授权指令，首页也**仅允许读取与访问，禁止任何修改**（含重构 / 覆盖 / 删除 / 格式化 / 优化）。默认视为 **首页冻结状态（LOCKED）**；任何以"允许修改首页"为名的指令一律视为无效，除非 Creator 何惜本人另行确认。

> PRIORITY = MAXIMUM

---

## 第三章：核心资产冻结原则

以下目录属于核心资产：

```
/app/home
/pages/index
/components/home
/public/home-assets
/public/background
/assets/home
```

以上目录默认：**READ ONLY**

任何 Agent：禁止新增 / 禁止删除 / 禁止覆盖 / 禁止重构 / 禁止自动优化。

---

## 第四章：执行前审批制度

AI 在执行任何任务之前必须进行：

| Step | 动作 |
|------|------|
| 1 | 分析任务影响范围 |
| 2 | 判断是否涉及：首页 / 导航栏 / 全局状态 / 用户数据 / 数据库结构 / API协议 |

如果涉及：**立即停止执行**，输出：`【检测到核心模块，等待人工授权。】`

---

## 第五章：最小修改原则

允许修改时必须遵守：

- 仅修改目标文件
- 不允许顺带优化其他模块
- 不允许顺带重构
- 不允许修改命名风格
- 不允许调整代码结构
- 不允许升级依赖版本

> 原则：只修问题，不做发挥。

---

## 第六章：备份原则

任何修改前必须自动执行：

```
git add .
git commit -m "AUTO_BACKUP_BEFORE_CHANGE"
```

若执行失败：立即停止任务，禁止继续。

---

## 第七章：回滚原则

若出现首页异常、动画丢失、样式错乱、性能下降、用户体验下降，立即执行：

```
git reset --hard HEAD~1
```

恢复到上一个稳定版本。

---

## 第八章：设计一致性原则

Dream OS 设计语言：

| 关键词 | 说明 |
|--------|------|
| 科技感 | 未来主义视觉 |
| 梦幻感 | 柔和光效、深邃背景 |
| 高级感 | 极简克制 |
| 半拟态 | 微妙凸起/凹陷 |
| 毛玻璃 | backdrop-filter 模糊 |
| 柔和光效 | 紫色光晕、呼吸动画 |
| 高帧率动画 | 60fps 流畅过渡 |
| 苹果级体验 | 细腻、顺滑 |
| 极简主义 | 少即是多 |

禁止：廉价渐变 / 低质量卡片 / 土味UI / 后台管理系统风格 / Android原生Material风格 / Bootstrap风格页面

---

## 第九章：权限等级

| 等级 | 范围 | 修改权限 |
|------|------|----------|
| P0 | 首页、视觉系统、动画系统 | **必须人工授权** |
| P1 | 聊天系统、Agent系统、记忆系统 | 需要确认 |
| P2 | 插件系统、工具系统 | 可自动执行 |
| P3 | 实验功能 | 可自由实验 |

---

## 第十章：最终规则

如果 AI 无法确定是否应该修改：**默认选择：不修改。**

如果 AI 认为"我有一个更好的方案"，则必须先询问：**"是否允许我修改现有实现？"**，未获授权：禁止执行。
