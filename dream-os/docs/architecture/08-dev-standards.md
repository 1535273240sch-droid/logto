# 08 - 开发规范

## 代码规范

1. 文件不超过 500 行
2. 单行不超过 120 字符
3. 禁止循环依赖
4. 模块间通过 EventBus 通信
5. Core 和 Services 层只读，不可修改

## 架构冻结规则

1. 冻结架构：先设计再开发
2. 冻结模块边界：模块间不直接调用
3. 固定通信方式：仅通过 EventBus + API
4. 组件化：所有 UI 使用可复用组件
5. 新增功能必须符合五层架构
6. 架构审查：每次变更经 Quality Guardian 审核

## 提交规范

1. 每次变更经 Quality Guardian 审核
2. 最多 3 次修复循环
3. 回归测试必须全部通过
4. 架构检查必须通过

## 测试规范

1. 17 个模块回归测试
2. 语法检查 (AST)
3. 文件大小检查
4. 架构侵入检查
